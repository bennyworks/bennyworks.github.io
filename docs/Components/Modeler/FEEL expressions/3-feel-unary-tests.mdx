一元测试是一种特殊的布尔表达式，应用于决策表的输入条目（即规则的条件），在满足以下条件之一时返回`true`：

- 当输入值应用于一元运算符时，表达式求值为`true`；
- 当输入值被赋给特殊变量 `?` 时，表达式求值为 `true`；
- 当表达式计算得到一个值，且输入值等于该值时；
- 当表达式计算得到一个列表，且输入值等于其中至少一个值时；
- 当表达式为`-`。

### 比较

将输入值与给定值进行比较。输入值隐式地作为操作符的第一个参数传递。

两个值必须是相同的类型。否则，结果为 `null`。

| 操作符 | 说明 | 支持类型 |
| --- | --- | --- |
| (none) | 等于 | 任何类型 |
| \< | 小于 | number, date, time, date-time, duration |
| \<= | 小于或等于 | number, date, time, date-time, duration |
| > | 大于 | number, date, time, date-time, duration |
| >= | 大于或等于 | number, date, time, date-time, duration |

```java
"valid"

< 10

<= date("2020-04-06")

> time("08:00:00")

>= duration("P5D")
```

### 区间

检查输入值是否在给定的`x` 和 `y`之间的一个区间内。

一个区间有两个边界，可以是开区间 `(x..y)` / `]x..y[` 或者闭区间 `[x..y]`。如果边界是闭合的，则包括给定的值（即小于等于/大于等于）。否则，不包括该值（即小于/大于）。

输入值隐式传递给操作符。

```java
// 输入 > 2 且 < 5
(2..5)

// 输入 > 2 且 < 5
]2..5[

// 输入 >= 2 且 <= 5
[2..5]

// 输入 > 2 且 <= 5
(2..5]
```

### 逻辑或

根据以下三元运算逻辑组合一元测试表达式：

- 如果至少有一个一元测试计算结果为`true`，则返回`true`；
- 否则，返回`false`。

```java
// 输入为 2 或 3 或 4
2, 3, 4

// 输入 < 10 或者 > 50
< 10, > 50
```

### 逻辑非

否定一个给定的一元测试表达式。这个表达可以是一个比较、一个区间或一个逻辑或。

如果给定的一元测试计算为`false`，则返回`true`。

```java
// 输入 != "valid"
not("valid")

// 输入 != 2 且 != 3
not(2, 3)
```

### 表达式

当单目运算符不足以表达条件时，可以使用任何返回布尔值的表达式，例如调用函数。

在表达式中，可以通过特殊变量 `?`访问输入值
